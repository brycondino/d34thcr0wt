Backend and reporting logic

This section shows how I would translate business reporting requirements into SQL, validation rules, and dashboard-ready datasets.

Metric Rules

Key report calculations for validation and dashboard trust.

Transportation:
Cost per KM = invoice_amount / actual_km
Budget Variance = actual_cost - contracted_cost
Empty Run % = empty_return_trips / total_trips

POS:
Net Sales = gross_sales - discount - refund
Gross Margin % = (net_sales - cost) / net_sales
Void Rate = voided_transactions / total_transactions

Sample SQL Logic — Transportation Expense Variance

SELECT
    s.route_id,
    s.origin,
    s.destination,
    c.carrier_name,
    SUM(fi.invoice_amount) AS actual_cost,
    SUM(s.actual_km * cr.rate_per_km) AS contracted_cost,
    SUM(fi.invoice_amount) - SUM(s.actual_km * cr.rate_per_km) AS variance_amount,
    ROUND(
      (SUM(fi.invoice_amount) - SUM(s.actual_km * cr.rate_per_km))
      / NULLIF(SUM(s.actual_km * cr.rate_per_km), 0) * 100, 2
    ) AS variance_pct
FROM shipments s
JOIN freight_invoices fi ON fi.shipment_id = s.shipment_id
JOIN carriers c ON c.carrier_id = s.carrier_id
JOIN carrier_contract_rates cr
  ON cr.carrier_id = s.carrier_id
 AND cr.route_id = s.route_id
 AND fi.invoice_date BETWEEN cr.effective_from AND cr.effective_to
GROUP BY s.route_id, s.origin, s.destination, c.carrier_name
HAVING ABS(SUM(fi.invoice_amount) - SUM(s.actual_km * cr.rate_per_km)) > 10000
ORDER BY variance_amount DESC;

Sample SQL Logic — POS Sales and Margin Summary

SELECT
    t.store_id,
    TRUNC(t.transaction_time) AS sales_date,
    COUNT(DISTINCT t.receipt_no) AS transaction_count,
    SUM(t.net_sales) AS net_sales,
    SUM(i.quantity * i.item_cost) AS total_cost,
    SUM(t.net_sales) - SUM(i.quantity * i.item_cost) AS gross_margin_amount,
    ROUND(
      (SUM(t.net_sales) - SUM(i.quantity * i.item_cost))
      / NULLIF(SUM(t.net_sales), 0) * 100, 2
    ) AS gross_margin_pct,
    ROUND(SUM(t.net_sales) / NULLIF(COUNT(DISTINCT t.receipt_no), 0), 2) AS avg_basket
FROM pos_transactions t
JOIN pos_items i ON i.receipt_no = t.receipt_no
WHERE t.transaction_time >= DATE '2026-06-01'
  AND t.transaction_time <  DATE '2026-07-01'
GROUP BY t.store_id, TRUNC(t.transaction_time)
ORDER BY sales_date, store_id;
